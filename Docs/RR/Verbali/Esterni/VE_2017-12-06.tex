	\subsection{Argomenti affrontati}
     Nel corso della riunione è stato approfondito lo scopo del progetto e sono state presentate e discusse le diverse tecnologie da apprendere per poterlo sviluppare.
     
			
	\subsubsection{Scopo del progetto}
	È stato concordato che lo scopo del progetto sarà quello di realizzare due plugin per Kibana per il monitoraggio di applicazioni. In particolare andrà sviluppata una mappa topologica e un plugin per la visualizzazione della lista delle trace dell'applicazione da monitorare. In particolare, per quanto riguarda la mappa topologica, è richiesta la possibilità di visualizzare i componenti dell'applicazione monitorata che possono essere server o database. Tali componenti devono essere rappresentati come nodi di un grafo.
	Inoltre è stata richiesta la possibilità di visualizzare sulla mappa le richieste che avvengono tra i componenti suddetti. Questa relazione deve essere rappresentata come archi di un grafo.
	Per quanto riguarda invece il plugin per la visualizzazione della lista delle trace ci sono state richieste le seguenti cose:
	\begin{itemize}
		\item{Deve essere possibile visualizzare il tempo di esecuzione, il momento in cui sono state eseguite le trace e lo status code con un eventuale messaggio di errore nel caso la richiesta HTTP fallisse;}
		\item{Deve essere possibile visualizzare un identificativo per ogni trace corrispondente alla richiesta HTTP;}
		\item{Deve essere possibile visualizzare il call tree relativo ad una determinata trace;}
		\item{In questo call tree deve essere possibile visualizzare il nome di ogni metodo presente, indentato in base alla sua "profondità";}
		\item{Deve essere possibile interagire con il call tree per agevolarne la comprensibilità;}
		
	\end{itemize}
	
	
	\subsubsection{Tecnologie}
     \begin{itemize}
     	\item { \textbf{Elasticsearch:} è stato presentata questa tecnologia che farà parte del progetto. Attraverso esso potremmo ottenere le informazioni dell'applicazione monitorata che andranno visualizzate. Sarà possibile ricercarle e raggrupparle mediante interrogazioni.
     	Sono state discusse le differenze che esistono tra Elasticsearch e i comuni database relazionale, ed è quindi stato spiegato in che modo le informazioni vergono organizzate all'interno di questo database.}
     	
     	\item { \textbf{Kibana:} è stata sucessivamente presentata questa piattaforma e ne sono state analizzate le diverse funzionalità.
     	Kibana è un tool open source progettato per rappresentare in modo grafico i dati presenti su Elasticsearch.
        L'obiettivo del nostro team sarà appunto quello di realizzare due plugin che permetta di visualizzare all'interno di questa piattaforma dei dati che riguardano il monitoraggio di un'applicazione.}
    
     
     \end{itemize}
